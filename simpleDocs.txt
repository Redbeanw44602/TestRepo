这里主要介绍仓库的结构设计，客户端如何访问仓库，以及如何正确设置仓库。

################
self.json
################

文件内容介绍：

format_version 100
name 显示在客户端仓库列表中的名称
identifier 设定后不可变动，仓库唯一uuid识别（镜像仓库也有独立的uuid）
status
	0 正常服务
	1 维护中
description 仓库简介
root 需要填写的字段
	groups[0] 资源组，客户端只能选定一个资源组
		name 资源组名称
		required_game_version 至少需要的游戏版本
		classes[0] 资源分类
			name 分类名称
			db_file 数据库下载链接
			description 简单介绍
			last_updated 最近更新时间戳
multi 其他功能
	SpeedTest 测速功能
		enable 是否允许客户端对本仓库主动测速
		file 测速文件下载链接
	PdbHashTable 用于判断版本的PDB-SHA1表
		enable 是否提供
		file 下载链接

其他介绍：

1. 链接
支持不同的协议，写法也较多样。
例：
	如root.groups[0].classes[0].name
	若不指定协议，写 test.db 将会在 <此仓库基链接>/groups/<GROUP_NAME>/test.db 下载文件
	也可以指定协议，
	目前支持 http(s), lanzou
	如 https://www.example.com/test.db
	或 lanzou://wqj8zhd，会直接下载该链接的文件

2. 测速
开启（推荐）有助于改善用户体验，而且对您的仓库利大于弊
利
	减少低速链接带来的资源浪费
	提升客户对您仓库的整体评价
弊
	如果你比较吝惜带宽资源（那其实你不应该公开仓库的）
	如果你的机器不能胜任巨大的请求量（那其实你不应该公开仓库的）

3. 链接缩写（如果缩写）
root.branches[0].resource	以   <此仓库基链接>/classes/	为基
multi.SpeedTest.file		以   <此仓库基链接>/multi/	为基
multi.PdbHashTable.file		以   <此仓库基链接>/multi/	为基
